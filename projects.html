Bits & Bytes

File Securer

Bits & Bytes is optimally used between two parties running the application on their respective devices. The algorithm it uses is derived from the AES-algorithm. A file can only be encrypted when a password is entered. The password forms part of the key used in the encryption process.

The user also has the option to delete the original file for extra safety measure.

After a file has been encrypted the user is prompted to select a location to save the encrypted file. when decrypting a file the user is prompted to select the encrypted file via the file explorer. The application checks if the file is indeed encrypted and if the password is correct.

If the conditions are met the file will be decrypted and the user will be prompted to once again choose a location to save the decrypted file and if they would like to delete the original.
